# 1. 概要

ruli（瑠璃）は Go に埋め込み可能な AST インタプリタ型スクリプト言語である。

設計思想：

- 静的型付け
- 明示的で安全な構文
- PLC / IEC61131 的な制御用途も視野
- Goホスト上で動作（GC依存でメモリ管理は行わない）
- まずASTインタプリタ（Phase1）、後にBytecode VMへ移行（Phase2）

# 2. 実行モデル

## 2.1 実行方式

- Go製ASTインタプリタ
- スクリプト単位で実行
- 各スクリプトは独立した環境（Env）を持つ

## 2.2 実行フェーズ

_Phase1: 宣言収集_

- type / func を先に登録
- シンボル解決可能にする

_Phase2: 実行_

- トップレベル文を上から実行

## 2.3 将来拡張

- Phase2で Bytecode VMへ移行可能（AST互換設計）

# 3. スコープモデル

## 3.1 スコープ階層

- Scriptスコープ（ファイル単位）
- Blockスコープ（{}）
- Functionスコープ

## 3.2 ルール

- トップレベルスコープはファイル内に閉じる
- ブロックスコープを持つ
- 関数スコープは独立
- 外側スコープ参照は可能（read/write）

## 3.3 変数探索ルール

代入時（=）：

- 現在スコープから検索
- 見つからなければ親へ
- Scriptトップまで探索
- 見つかった最初の変数を更新
- 存在しなければエラー

## 3.4 宣言ルール

- := / type 宣言は現在スコープに作成
- 再宣言はシャドーイング可能

## 3.5 シャドーイング

許可する：

```
x := 10

if true {
  x := 20
}
```

# 4. 型システム

## 4.1 基本型（v1）

- INT
- REAL
- BOOL

STRINGはv1では未サポート（後日検討）

## 4.2 distinct type

```
type TEMP INT
```

- 新しい型として扱う
- 基本型との暗黙変換なし

## 4.3 struct

```
type Motor struct {
  speed:  INT
  enable: BOOL
}
```

- フィールドは公開
- 値渡し

## 4.4 配列

固定長配列のみサポート

```
values: [10]INT
matrix: [3][3]INT
```

_特徴_

- 型の一部としてサイズを持つ
- 多次元配列対応
- 値渡し（コピー）

## 4.5 配列初期化

```
a := [3]INT{1, 2, 3}
```

## 4.6 添字アクセス

```
a[0] = 10
x := a[1]
```

## 4.7 境界チェック

- 実行時チェックあり
- 範囲外アクセス → runtime error

## 4.8 len()

組み込み関数：

```
len(array)
```

- 配列の要素数を返す
- コンパイル可能なら定数化可能 5. 関数

## 5.1 定義

```
func add(a: INT, b: INT) INT {
  return a + b
}
```

## 5.2 引数

- 完全値渡し

## 5.3 戻り値

- 複数戻り値対応

```
func div(a: INT, b: INT) (INT, BOOL)
```

## 5.4 呼び出しルール

- 戻り値は無視可能

```
div(10, 2)
```

- 複数戻り値を単一変数へ代入は禁止

```
x := div(10, 2) // error
```

正しくは

```
x, ok := div(10, 2)
```
